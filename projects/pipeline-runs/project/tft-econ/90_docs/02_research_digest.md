## Key Facts

**事实 1（强证据）**：金铲铲之战核心经济参数——

| 参数 | 说明 | 备注 |
|------|------|------|
| 基础收入 | 每回合固定 5 金币 | 不变 |
| 利息 | 每 10 金币存款 → 1 利息，上限 5（存款 50 封顶） | 核心经济杠杆 |
| 连胜奖励 | 2连胜+1, 3连胜+1, 4连胜+2, 5连胜+3 | ⚠️ 需验证当前版本 |
| 连败奖励 | 与连胜对称或略不同 | ⚠️ 需验证 |
| PVE 回合 | 打野回合收入可能不同 | 需确认 |

> ⚠️ **参数需建模前确认当前版本精确值**。版本更新频繁，建议明确基于哪个赛季。

**事实 2（强证据）**：利息系统是最核心的经济杠杆——
- 存款 0→50，每 10 金币多 1 利息
- "不花钱"的回报率 10%/回合（到 50 金币后边际为 0）
- **关键权衡**：花 10 金币升级/刷新 → 损失每回合 1 金币利息 → 需评估战力提升是否值得

**事实 3（强证据）**：连胜/连败策略的数学本质——
- **连胜**：保持赢 → 连胜奖励 + 高血量 → 但需投入金币维持战力
- **连败**：故意输 → 连败奖励 + 选秀优先权 + 不投金币 → 利息积累更快 → 后期爆发
- **最优策略**在"连胜收益"和"连败+利息复利"之间找平衡

**事实 4（强证据）**：数学建模框架——

**推荐：动态规划（DP）**
- **状态**：(当前回合, 当前金币, 当前连胜/连败数, 当前等级)
- **决策**：每回合"花钱刷新/升级" vs "存钱吃利息"
- **目标**：最大化特定时点的期望战力/排名
- **约束**：金币 ≥ 0

**备选：蒙特卡洛模拟**
- 模拟大量对局，统计不同策略的平均排名
- 可纳入随机因素（对手匹配、选秀）
- 不给出解析最优解，只给统计结论

**建议两者结合**：DP 给理论最优，蒙特卡洛验证鲁棒性。

**事实 5（中等证据）**：论文级文档框架——

| 章节 | 内容 |
|------|------|
| 摘要 | 问题、方法、核心结论 |
| 1. 引言 | 背景、研究意义、相关工作 |
| 2. 经济系统建模 | 参数定义、收入公式、状态空间 |
| 3. 策略分析 | 连胜/连败/混合策略的数学推导 |
| 4. 最优策略 | DP 求解、蒙特卡洛验证、结果分析 |
| 5. 可视化与验证 | 交互工具说明、敏感性分析 |
| 6. 结论与展望 | 发现、局限、未来方向 |

**事实 6（中等证据）**：可视化核心维度——
1. **经济曲线图**：不同策略下金币随回合变化
2. **策略对比热力图**：不同起始条件下的最优策略
3. **参数敏感性**：调整参数后最优策略如何变化
4. **模拟器**：输入初始条件 → 运行 → 输出结果
5. **动态演示**：逐步展示经济决策过程

**事实 7（中等证据）**：可视化技术方案——
- **图表库**：ECharts（中文生态好、交互能力强、支持热力图/动态更新）> Chart.js（轻量）> D3.js（灵活但成本高）
- **推荐 ECharts**
- **框架**：Vue3 + Vite 或纯 HTML + ECharts CDN
- **部署**：GitHub Pages / Vercel

**事实 8（弱证据）**：金铲铲之战与 TFT 国际版可能有参数差异——核心机制一致，但版本节奏和部分数值可能不同。**建议明确建模基于哪个版本。**

## Key Cases

### 参考

**案例 1：TFT 经济分析社区研究**（Reddit/Bilibili）
- 常见结论："50 金币前存钱最优"、"连败特定版本优于连胜"
- **启示：系统化这些经验，用数学模型验证其正确性**

**案例 2：运筹学 Stochastic Dynamic Programming**
- TFT 经济本质是随机动态规划（战斗结果有随机性）
- 类似库存管理、投资组合优化
- **启示：借鉴运筹学标准解法框架**

**案例 3：游戏经济模拟器可视化先例**
- LOL/Dota 经济模拟项目，折线图展示策略收益曲线
- **启示：核心是"策略对比"——让用户直观看到不同选择的长期后果**

### 反证

**案例 4：版本过时导致建模无效** → 旧版本参数在新版本中失效。**必须标注版本号 + 参数可配置。**

**案例 5：忽略随机性的"伪最优"** → 纯 DP 假设战斗结果确定，现实不可达。**必须蒙特卡洛验证鲁棒性。**

**案例 6：文档过于学术导致不可读** → 可视化网站面向普通玩家，语言应通俗。**文档学术严谨 + 可视化通俗易懂。**

## Industry Norms

1. **游戏经济建模**：参数定义 → 公式推导 → 状态空间 → DP/模拟求解 → 敏感性分析
2. **策略分析**：枚举策略空间 → 期望收益计算 → 最优 → 鲁棒性验证
3. **学术文档**：摘要 → 引言 → 建模 → 求解 → 验证 → 结论
4. **可视化**：核心图表 + 参数可调 + 实时更新 + 导出
5. **中文游戏术语**：连胜/连败/利息/D/刷/吃利息/运营/经济管理

---

## Failure Cases

1. **版本依赖**：硬编码参数 → 版本更新后快速过时
2. **过度简化**：忽略选秀优先权、血量隐性资源 → 偏差大
3. **过度复杂**：纳入所有因素 → 无法求解
4. **文档不成体系**：有公式无推导，有结论无验证
5. **可视化无交互**：静态图表 → 用户无法探索参数空间
