## Problem Definition

**核心问题：** TFT/金铲铲之战玩家需要一个能系统性理解连胜/连败策略经济权衡的工具——通过数学建模验证直觉，通过交互可视化探索参数空间，最终回答"什么条件下哪种策略更优"。

**Brief 质量评估：** 一行 Goal 信息密度较高（主题、方法、核心问题、双交付物），但缺少 Target User、Scope、Success Criteria、Constraints。pm-core 自行补全。

**关键判断：**
- **网站定位：** 独立的策略探索工具（而非论文附属品）。独立工具用户覆盖面更广，体验设计不受论文结构束缚。
- **研究方法：** 策略对比 + 蒙特卡洛模拟，而非严格最优解证明。多阶段博弈问题严格最优解不可达，策略对比 + 模拟对目标受众更有价值。
- **文档深度：** 结构严谨的技术分析文档，非学术论文。
- **建模范围：** 硬性限定纯经济变量（基础金币 + 利息 + 连胜/连败奖励），排除 D 牌概率、阵容构建、对手行为。
- **开发顺序：** 先文档（建模 + 模拟）后网站（可视化 + 交互）。
- **可视化交互：** 参数调整 < 200ms 即时反馈。必须支持多策略曲线叠加对比 + hover 经济明细。

---

## Target User

**核心用户：** TFT/金铲铲之战中度到重度玩家——理解基本经济机制，但不确定"什么时候该攒钱、什么时候该花钱"，需要比经验法则更精确的策略指导。

**次要用户：** 对游戏经济系统感兴趣的数学爱好者 / 运筹学学习者。

**使用环境：** 桌面浏览器，本地运行或静态部署（GitHub Pages / Vercel）。

## Core Scenarios

### 场景 1：策略对比探索
用户打开网站，看到 4 种预设策略的金币积累曲线叠加显示。调整"回合数"滑块，所有曲线实时更新。鼠标悬停第 10 回合，看到每种策略的经济明细（基础收入、利息、连胜/连败奖励、总金）。

### 场景 2：参数敏感性分析
用户修改"连胜奖励倍率"参数，观察连胜策略曲线是否仍优于连败。发现连败策略从第 8 回合开始超越。

### 场景 3：最优策略推荐
用户设置参数后点击"推荐最优策略"。系统模拟 1000 次，输出"连败转连胜策略期望经济最高，比纯攒利息多 15 金"。

### 场景 4：阅读技术分析文档
用户打开文档，从问题定义读到结论，通过文档中的链接跳转到网站验证结果。

## Core Features

### F1：经济数学模型

**建模范围（硬性约束）：**

| 纳入变量 | 说明 |
|---|---|
| 基础收入 | 每回合固定 5 金币 |
| 利息机制 | 每 10 金币 → 1 利息，上限 5（存款 50 封顶） |
| 连胜奖励 | 2 连胜 +1, 3 连胜 +1, 4 连胜 +2, 5 连胜 +3 |
| 连败奖励 | 与连胜对称或略不同（需确认当前版本） |

| 排除变量 | 理由 |
|---|---|
| D 牌概率 | 涉及随机过程 + 组合优化，复杂度不可控 |
| 阵容构建 | 超出经济系统范围 |
| 对手行为 | 博弈论层面不可控 |
| 血量隐性资源 | 难以量化 |
| 选秀优先权 | 难以量化 |

**数学方法：** 动态规划（DP）理论分析 + 蒙特卡洛模拟验证。

**状态空间：**
- 状态：(当前回合, 当前金币, 当前连胜/连败层数)
- 决策：每回合"花钱" vs "存钱吃利息"
- 目标：最大化 N 回合末期望经济值

**模型输出：** 每种策略在每回合的期望金币分布（均值 + 方差）。

### F2：技术分析文档

**格式：** Markdown + LaTeX 公式，有目录、公式编号、图表引用。非学术期刊标准，论证严谨。

**章节结构：**

| 章节 | 内容 |
|---|---|
| 摘要 | 问题、方法、核心结论（300 字内） |
| 1. 引言 | TFT 经济系统背景、研究动机、核心问题 |
| 2. 经济系统建模 | 参数定义、收入公式、状态空间、决策变量 |
| 3. 策略定义 | 4 种典型策略的严格定义和公式 |
| 4. 数学分析 | DP 推导、各策略的理论期望收益 |
| 5. 模拟验证 | 蒙特卡洛模拟设计、结果分布、策略对比 |
| 6. 结论 | 什么条件下哪种策略更优、核心洞察 |
| 附录 | 参数表、完整模拟数据 |

**一致性要求：** 文档公式和网站曲线必须是同一模型输出，不允许不一致。

### F3：4 种典型策略

| 策略 | 定义 |
|---|---|
| S1：纯攒利息 | 不花钱，快速攒到 50 金封顶吃满利息 |
| S2：纯连胜 | 前期积极投入金币维持连胜 |
| S3：连败转型 | 前期故意不投资（接受连败 + 利息积累），中期转型投入 |
| S4：均衡策略 | 保持一定金币底线（如 30 金），适度投资维持中等排名 |

### F4：交互可视化网站

**布局：** 单页应用，上方参数面板 + 下方图表区。

**参数控制面板（3-5 个核心参数）：**

| 参数 | 类型 | 默认值 | 范围 |
|---|---|---|---|
| 回合数 | 滑块 | 30 | 10-50 |
| 初始金币 | 滑块 | 10 | 0-50 |
| 连胜/连败奖励方案 | 下拉选择 | 默认方案 | 方案 A / B（可扩展） |
| 模拟次数 | 滑块 | 1000 | 100-10000 |
| 利息策略阈值 | 滑块 | 50 | 10-50 |

**图表区：**

| 图表 | 类型 | 说明 |
|---|---|---|
| 策略收益曲线 | 折线图（X=回合，Y=累计金币） | 4 策略叠加，不同颜色 |
| 收益分布对比 | 箱线图 / 柱状图 | 各策略末回合金币分布 |
| 最优策略高亮 | 自动标注 | 当前参数下最优策略 |

**交互要求：**
- 参数调整后所有图表 **< 200ms** 更新（纯前端同步计算）
- Hover 曲线任意点 → 经济明细 tooltip（基础收入、利息、连胜/连败奖励、总金）
- 支持"理论期望"和"运行模拟"两种视图
- 图例点击切换显示/隐藏策略曲线

### F5：最优策略推荐

- 基于当前参数，蒙特卡洛模拟 N 次
- 输出结论卡片："当前参数下，X 策略最优，比最差策略多 Y 金（±Z）"
- 在曲线上高亮最优策略

## Key Trade-offs

**取舍 1：策略对比 + 模拟 vs 严格最优解** — 多阶段博弈问题严格最优解不可达。4 种策略 + 蒙特卡洛模拟在合理时间内给出有结论的输出。代价：不保证全局最优。

**取舍 2：纯经济变量 vs 完整经济系统** — 纳入 D 牌/阵容/对手后建模无限膨胀。纯经济变量可解、结论可交付。代价：结论限于纯经济视角。

**取舍 3：技术文档 vs 学术论文** — 学术论文投入 10 倍以上。技术文档论证严谨但不追求发表标准。代价：不可投稿学术期刊。

**取舍 4：先文档后网站** — 网站内容依赖模型输出。先文档后网站开发可控。代价：两个交付物不能并行。

**取舍 5：纯前端同步计算** — 1000 次 × 30 回合 = 3 万次简单运算，毫秒级。不需要 Web Worker。

## Handoff to architect

1. **项目结构：** `model/`（数学模型 + 模拟引擎，纯 JS 模块）、`docs/`（Markdown 文档）、`web/`（交互可视化）。

   ```js
   simulate(config, strategyName, rounds, simulations) → [{ mean, std, details }]
   theoreticalExpectation(config, strategyName, rounds) → [{ mean, details }]
   ```

4. **参数可配置：** 连胜/连败奖励梯度、基础收入、利息上限等从配置对象读取。支持版本切换。

5. **性能：** 参数调整 < 200ms 更新所有图表。1000 次 × 30 回合同步计算 < 100ms。

6. **图表：** 折线图（策略曲线叠加）+ 箱线图/柱状图（模拟分布）+ tooltip（hover 经济明细）+ 图例切换。

8. **不可被架构覆盖的体验：**

   - 以上如无法实现，**退回 pm-core**
