# TFT 经济模拟器 — 技术分析文档

## 1. 问题定义

TFT（金铲铲之战）玩家面临一个核心经济决策：每回合多少金币投入阵容，多少金币留存获取利息。本系统通过数学建模和蒙特卡洛模拟，系统性分析 4 种经典策略的经济表现，回答"什么条件下哪种策略更优"。

## 2. 建模范围

### 纳入变量
| 变量 | 值 | 说明 |
|---|---|---|
| 基础收入 | 5 金/回合 | 固定 |
| 利息 | floor(金币/10)，上限 5 | 每 10 存款 = 1 利息 |
| 连胜奖励 | 0/0/1/1/2/3 (0-5 连胜) | |
| 连败奖励 | 与连胜对称 | |

### 排除变量
- D 牌概率、英雄池、阵容构建
- 对手行为、选秀机制
- 血量 → 经济损失的复杂映射

## 3. 策略定义

| 策略 | 描述 | 花费模式 | 经济特征 |
|---|---|---|---|
| **纯攒利息** | 不花钱，全攒利息 | 0 金/回合 | 利息最大化 + 连败奖励 |
| **纯连胜** | 花金币保赢 | 4→3→2 金/回合 | 连胜奖励 + 低利息 |
| **连败转连胜** | 前 4 回合攒金，后转胜 | 0→3→2 金/回合 | 先攒后花，折中 |
| **均衡** | 交替胜负 | 0/4 交替 | 无稳定收益 |

## 4. 模拟结果

### 参数配置
- 回合数：20
- 基础收入：5 金
- 利息上限：5
- 连胜/连败 ×5 奖励：3 金
- 蒙特卡洛次数：1000

### 结果

| 策略 | 期望金币 | 中位数 | Q25-Q75 |
|---|---|---|---|
| 纯攒利息 | **226.3** | 226 | 224-229 |
| 连败转连胜 | 184.0 | 184 | - |
| 均衡 | 179.8 | 181 | - |
| 纯连胜 | 162.4 | 162 | - |

**结论：纯攒利息策略期望经济最高，比纯连胜策略多 63.9 金（约 39%）。**

### 分析

1. **纯攒利息优势来源**：不花费金币 → 金币全部积累 → 利息收益 + 连败奖励双重叠加
2. **纯连胜劣势来源**：每回合花费 2-4 金币 → 金币基数低 → 利息收益大幅下降，连胜奖励无法弥补
3. **连败转连胜的折中价值**：前期攒金（0 花费）建立利息基础，后期适度花费维持连胜，但转胜后的花费拖累了总收益
4. **均衡的随机损失**：随机胜负导致连胜/连败奖励不稳定，且有时花费金币却仍输

## 5. 性能指标

- 1000 次 × 30 回合 < 10ms（纯 JS 前端同步计算）
- 参数调整即时反馈（< 200ms 目标）

## 6. 模型局限性

1. **胜负概率简化**：实际胜负受对手、阵容影响，本模型用花费阈值简化
2. **无血量经济映射**：连败策略实际上会损失血量，本模型未纳入
3. **策略固定化**：实际游戏中策略应根据对手动态调整
4. **参数可调性**：不同 TFT 版本的连胜/连败奖励梯度不同

## 7. 验证方法

模型可通过以下方式验证：
1. 在网站上调整参数观察曲线变化
2. 修改 `config.js` 中的参数测试不同版本
3. 运行 `model/test.js` 查看详细输出
