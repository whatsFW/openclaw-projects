# 决策记录：wfood 外卖模拟平台

## D1：技术栈选型 — Vue3 + Vite + Pinia + Vue Router

- **决定：** 使用 Vue3 + Vite + Pinia + Vue Router
- **原因：** 用户原始需求指定 Vue3+Vite；Pinia 是 Vue3 官方推荐状态管理；Vue Router 是标准路由方案
- **影响：** 所有源码基于此技术栈，后续扩展需保持一致
- **状态：** ✅ 已落地

---

## D2：API 层统一抽象 — 当前 Mock，预留后端接口

- **决定：** `src/api/` 下按模块拆分（user.js、shop.js、service.js），当前返回 Mock 数据，后续切换真实后端只改实现不动业务代码
- **原因：** 产品要求"纯前端但预留后端接口"，通过 API 抽象层实现关注点分离
- **影响：** 业务组件（views/）直接调用 API 层函数，不感知 Mock/真实切换
- **状态：** ✅ 已落地

---

## D3：购物车状态管理 — Pinia + localStorage 双层

- **决定：** 购物车用 Pinia CartStore 管理，通过 `$subscribe` 自动同步到 localStorage；切换店铺时自动清空购物车
- **原因：** 购物车需要跨页面共享 + 刷新不丢失 + 店铺隔离（外卖场景下不同店铺不能混单）
- **影响：** 店铺详情页 `onMounted` 需检查 `shopId`，不匹配则 `clearCart()`
- **状态：** ✅ 已落地

---

## D4：拟物化 UI — CSS 变量 Design Token

- **决定：** 拟物化风格通过共享 CSS 变量（`src/assets/tokens.css`）统一控制，按钮/输入框/卡片/导航栏统一样式
- **原因：** 产品要求"拟物化 UI"，需要一套统一的视觉规范，避免各页面风格不一致
- **影响：** 所有组件引用同一套 Design Token，调整风格只需改 tokens.css
- **状态：** ✅ 已落地

---

## D5：页面结构 — 6 个核心页面

- **决定：** Login → Register → ShopList → ShopDetail → Cart → OrderConfirm
- **原因：** 覆盖完整外卖流程（登录注册 → 浏览 → 选品 → 购物车 → 下单）
- **影响：** 路由按此结构配置，每个页面对应一个 .vue 文件
- **状态：** ✅ 已落地
